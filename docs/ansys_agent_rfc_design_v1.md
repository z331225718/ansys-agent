# 📝 项目技术白皮书：面向 Ansys AEDT 的电磁仿真智能体 (ansys-agent)

## 1. 核心愿景与项目定位

### 1.1 最终愿景：全职电磁仿真“数字员工”
打造一个具备领域专家知识的 AI Agent，能够深度理解电磁仿真业务需求。其终极形态是能够从白纸（0）开始进行拓扑选型、3D建模、边界条件敷设，并自动闭环优化至最终交付标准（100）。

### 1.2 现阶段务实定位：聚焦“1 到 100”的黄金价值区间
鉴于当前 Ansys AEDT 尚未提供完全面向大模型的通用全功能 CLI，且大模型在精准控制本地图形界面（GUI）上存在天然的“信息与控制断层”，**项目当前的核心任务是彻底接管“1 到 100”的自动调谐与迭代优化阶段。**
* **人类工程师（0 -> 1）：** 负责空间拓扑抽象、在 AEDT 中建好基础模型、定义核心参数变量（如宽度 $W$、长度 $L$）、设置基础边界条件与网格剖分。
* **AI 数字员工（1 -> 100）：** 负责承接指标需求，通过多轮仿真驱动、曲线自动评估、专家策略调参，直至模型完全达标。

> 💡 **核心价值定位**：释放工程师在常规、重复的“改参数 - 等仿真 - 看曲线 - 再改参数”循环中的高昂时间成本。在工业界，“1 到 100”的自动化才是压榨生产力、最值钱的核心部分。

---

## 2. 核心技术架构设计

项目采用**“高确定性控制树”**架构，拒绝让大模型裸写代码，确保工业仿真过程的绝对安全与可控。

```
[ 用户 / 高级工作台 ] -> 输入目标指标 (如: S11 < -20dB @ 2.4GHz)
        |
        v
[ LLM Planner (核心大脑) ] -> 生成结构化 Workflow JSON
        |
        v
[ Backend Validator (校验层) ] -> 检查高危命令 / License 状态 / 语法合规性
        |
        v
[ Controlled NodeExecutor (执行层) ] -> 异步驱动本地 PyAEDT / 真实 AEDT 2026.1
        |
        v
[ 仿真运行与数据提取 ] -> 自动导出 Touchstone (.s2p) / 曲线图片 / 场强图
        |
        v
[ VLM / 规则评估器 ] -> 评估指标是否达成？
        |-- (未达成) -> 携带 Traceback / 曲线图片 -> 进入 [Repair Loop] 重新规划
        |-- (已达成) -> 输出完美模型与 Report.html -> 交付工程师评审
```

## 3. 三大核心痛点突破策略（讨论焦点）

### 3.1 痛点一：大模型“看不到”仿真曲线与图形界面
* **解决方案：** 引入视觉语言模型（VLM）弥补视觉缺陷。
* **实施路径：** 在仿真跑完生成 Report 后，调用 PyAEDT 导出图片接口（如 `orpt.export_report_to_jpg()`），截取 S 参数曲线图或场强图。将图片连同数值数据作为 `Context` 喂给支持 Vision 的模型（如 Claude 3.5 Sonnet / GPT-4o），利用 VLM 极强的图形识别能力替代工程师的眼睛来识别“意料之外的谐振点”。

### 3.2 痛点二：如何做“专家级调参”而非盲目的数学优化
* **解决方案：** 参数化状态机 + 物理先验提示词（Physics-informed Prompts）。
* **实施路径：** AI Agent 的定位不是替代 AEDT 自带的 Optimetrics（局部数学优化器），而是进行**全局专家级策略调整**。通过在 System Prompt 中注入电磁学与微带线原理知识（例如：“若带外抑制不够说明谐振零点偏移，应增大 $L_2$ 的长度，或减小间距”），让 AI 像真正的初级工程师一样进行有逻辑、有策略的协同试错调参。

### 3.3 痛点三：脚本报错与仿真异常导致自动化中断
* **解决方案：** 打造强健的 Repair Loop（错误自动修复循环）。
* **实施路径：** 在设计中引入 `max_repair_attempts: 3` 机制。当本地执行器发生几何干涉、NameError、或 AEDT 提示 License 溢出等报错时，将完整的 Traceback 运行时错误日志捕获并回传，利用大模型的高级推理与 Debug 能力，在线重新规划并修正 Workflow JSON，实现无人值守的自动重试。

## 4. 借鉴与魔改开源项目（如 Pi 框架）的路线图

为了避免重复造轮子，项目将深度参考并“吸骨榨髓”像 **Pi (earendil-works/pi)** 这样优秀的本地/终端 Agent 框架，执行**“去粗取精”**的魔改与重构方案。

### 4.1 彻底剥离并吸收的“灵魂”组件
1. **RPC / Session 隔离机制：** 剥离 Pi 的终端 TUI 界面和前端高亮代码，将其优秀的多会话管理、明晰的 Session ID 控制和长连接能力封装进项目的 `api/` 路由中，作为纯粹的后端 Agent 引擎。
2. **Extension 强类型工具注册：** 学习 Pi 注册本地工具的方式。把针对 PyAEDT 编写的高频核心函数（如 `CreateAirBox`、`SetPort`、`ChangeVariable`）标准化为 Agent 的专用 Tools 库，使大模型能够安全组合这些 100% 可行的高可靠性脚本。
3. **Project Trust（项目信任机制）：** 保留安全沙箱和权限白名单机制。在 `v0.79.1` 中 Pi 进一步优化了默认项目信任设置，这能严格锁死 Agent 的文件系统访问权限，防止 AI 误操作导致删除本地关键仿真资产。

### 4.2 果断抛弃的“杂质”组件
* **通用型 Agent 的提示词包袱：** 清空 Pi 内置的通用日常聊天、写普通代码的提示词，全面替换为纯粹的电磁仿真专家 System Prompt。
* **繁重的终端交互代码：** 砍掉 Pi 所有的终端命令行自动补全触发器等上层建筑，完全由我们自己的 Web 高级工作台（`/advanced` 页面）和独立的 Benchmark 系统替代。

## 5. 项目进化路线图 (Milestones)

* **Stage 1: 最小可行性闭环 (MVM) —— 彻底打通 1->100**
  * **目标：** 针对一个固定拓扑（如：特定频段的微带贴片天线或功分器），实现“输入目标指标 -> 自动多轮调参 -> 仿真验证 -> VLM/文本判断 -> 最终达标”的无人值守全自动闭合回路。
* **Stage 2: 模板化裂变 —— 蚕食 0->1 边界**
  * **目标：** 构建 `workflow_templates/` 库。用户只需说出需求，AI 自动检索并加载“切比雪夫滤波器”基础模板，通过参数化代码在后台直接把 0 到 1 的初始几何体、空气盒自动画出来，实现形式上的从 0 到 1。
* **Stage 3: 终极形态 —— 真正意义的 0->100 数字员工**
  * **目标：** 随着 PyAEDT API 走向成熟，AI Agent 能够直接阅读 PDF 规格书（Spec），自行在大脑里推导出合理的 3D 拓扑，从 Blank Project 开始一行行生成完美的建模、边界与仿真脚本。
