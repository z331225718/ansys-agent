边界

已跑通 3 个真实 AEDT smoke，分别覆盖 geometry/setup/sweep、select_face/port、airbox/boundary。BRD/MCM import-cutout 仍是 experimental track，默认不进入 HFSS core catalog；当前路径是 model-build only，资源受限机器上默认不 analyze。validation 仍是结构性检查，不等价于完整电磁物理正确。chat planner 当前是确定性骨架，后续可接主模型，但输出仍必须限制为 workflow JSON。node evolution 只生成 proposal，不自动发布 stable 节点。

BRD Experimental Track

BRD/MCM workflow 面向真实高频板级场景：导入 Cadence BRD/MCM、解析 net、PyEDB cutout、导入 stackup、定位端口候选、创建 3D Layout port 和宽频 setup。当前产物重点是可复用 layout service、明确的 experimental 节点 contract、MCP fake/real smoke 入口，以及可人工检查的 AEDT model-build artifact。
